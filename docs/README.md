## 변경 이력

- **v1.0** — 대표 미팅 반영 초안 (단일 파일 `prd-v1.md`)
- **v1.1** — 인증·약관, 대시보드, 알림 시나리오, 결제 부가기능, 예약 상태 전이, 상담 링크 정책, 고객지원, 기술 스택 섹션 보완
- **v1.1 split** — 단일 파일을 13개 문서로 분리, 폴더 구조로 정리
- **v1.1 pricing** — 고정가(29,900원)에서 **멘토 자율 책정 + 최소금액 29,900원 floor** 모델로 전환 (인프런형). 신규 멘토는 후기/완료 5건까지 floor 락, 가격 변경 주기 주 1회 제한
- **v1.2 pricing (2026-05-04)** — **강제 floor(₩29,900) 폐지 + 신규 멘토 floor 락 폐지**. 관리자 설정 `price_min`/`price_max` 범위 내 멘토 자율 조정으로 전환. ₩29,900은 UI 권장가(기본값)로만 유지. `platform_settings` 테이블 도입 (ADR-003)
- **v1.3 legal (2026-05-04)** — 핵심 약관 본문 초안(T01·T02·T05·T09·T13) 신규 문서화(docs/19). ADR-004: 만 19세 미만 결제 민법 §5 고지+확인 체크박스 추가. 전자상거래법 청약철회 예외 적용 기준 명시(§18-12). 통신판매중개업자 신고 요건 정리(§18-13).
- **v1.1 auth flow** — 가입 플로우 순서 명문화 (소셜 → 역할 → 약관 → 본인인증 → 정보입력). 약관은 **역할별 세트**로 분리: 멘토는 서비스 운영/정산/과세/행동 가이드 4종 추가 필수
- **v1.1 mentor reliability** — 멘토 가능시간 리드타임 7일, 일정 변경 제안 72h 이전까지, 누적 경고 기반 곱 증가 패널티(5→10→20→40→80%) 체계 확정. 사유 드롭다운 5종 필수화
- **v1.1 enum catalog** — 전 프로젝트 enum을 docs/14에 중앙화. DB CHECK·Java enum·TS union type의 단일 Source of Truth
- **v1.1 T1 policies** — 9건 핵심 정책 확정: 노쇼 10분 / 정산 주간 / ~~가격 floor 29,900~~ / ~~floor 락 "후기 5 AND 완료 5"~~ / 14세 미만 동의 UX / 개인정보 5년·3년·3개월 3단 보관 / 멘티 탈퇴 절차 / 결제 30분 내 3회 재시도 / 멘티 노쇼 5단계 제재 (floor 관련 2건은 v1.2에서 재확정)
- **v1.1 T2 policies** — 11건 UX·분쟁 예방 정책 확정: 후기 14일 기한·7일 수정·30일 삭제·멘토 답글·신고 / 상품 수정·삭제 규칙 / 프로필 수정 재심사 분기 / 멘토 탈퇴 / 멘토 휴가 `ON_LEAVE` 최대 30일 / 역할 겸용 허용 / 자녀 프로필 5명 / 프로필 공개 범위(이니셜 기본) / 녹화 원칙 금지 + 동의 시 허용 / 외부 연락처 금지 + 3단계 제재
- **v1.1 simplification** — 알림 14종 → **알림톡 8종 + 이메일 2종**으로 축소. 자동 확정 모델 도입 (멘토 수락/거절 단계 제거, `REJECTED` 상태 제거). 사전질문지: 멘토가 미리 작성 → 멘티 결제 시점 답변 강제. Zoom API 자동 생성은 P1 백로그
- **v1.1 legal/tax compliance** — 종이 계약서 없이 약관 동의 기반 정산 운영을 위한 §18-11(멘토-플랫폼 위탁 관계 7종 조항), §21-7(원천징수 정산 흐름 사업소득 3.3% 가정), §13-7(법무·세무·노무 3종 자문 체크리스트) 추가. Day 0에 세무사·노무사 자문 트랙 신설

## 최종 의사결정 문장

본 서비스는 기존 오프라인/캠퍼스투어 중심 멘토링 구조에서 벗어나, 학생과 학부모의 실제 고민 해결에 초점을 둔 **온라인 멘토링 중개 서비스**로 재정의한다. MVP 단계에서는 **3개 대학과 단과대 기반 구조**를 중심으로 멘토 대표성을 확보하고, **관리자 설정 범위 내 자율 가격(권장가 29,900원)의 30분 멘토링 상품**을 중심으로 검증된 멘토 프로필, 예약, 결제, 카톡 알림, 후기, 정산 기능을 우선 구현한다. 플랫폼은 패키지 운영보다 중개에 집중하며, 운영 리스크를 낮추면서 초기 전환율과 수요를 검증하는 방향으로 진행한다.
