# 06. 알림 시스템

## 11-6. 알림 시스템

### 채널 정책

- **기본 채널 (거래성)**: 카카오 알림톡 (NHN Toast Bizmessage 또는 알리고 등)
- **폴백**: 알림톡 수신 실패 시 SMS/LMS 자동 전환
- **이메일 (보조 + 시스템성)**: 정산·영수증 / 환불 통지 / 멘토 승인·반려 / 분쟁 증빙용 통지
- **후속 확장**: 앱 푸시, 자체 알림 시스템

### 알림톡 템플릿

자동 확정 모델 기반 (멘토 수락 단계 없음). 결제 완료 시점에 양측에게 A02 단일 확정 알림 발송.

| 코드 | 트리거                      | 수신자 | 우선순위 | 비고                                          |
| ---- | --------------------------- | ------ | -------- | --------------------------------------------- |
| A02  | **결제 완료 / 예약 확정**   | 양측   | MVP      | 자동 확정 모델: 멘티 결제 즉시 멘토에게도 발송 (예약 신규 안내 + 확정) |
| A03  | 멘토 상담 링크 등록 완료    | 멘티   | MVP      | 마이페이지 상담 입장 안내                     |
| A04  | 3일 전 리마인드             | 양측   | MVP      | 확인 버튼 포함, 링크 미등록 시 멘토 강조      |
| A05  | 당일 리마인드               | 양측   | MVP      | 확인 버튼 포함                                |
| A06  | 상담 시작 10분 전 입장 안내 | 양측   | MVP      | 링크 미등록 시 SMS 폴백 + 관리자 에스컬레이션 |
| A07  | 상담 종료 후 후기 요청      | 멘티   | MVP      | 종료 1시간 후                                 |
| A08  | 정산 완료                   | 멘토   | MVP      | 정산 상세 링크                                |
| A01  | 일정 변경 제안              | 상대방 | Week 5+  | 동의/거절 액션 (멘토 → 멘티 / 멘티 → 멘토)   |

### 이메일 (MVP 2종)

알림톡 심사 기간 중 폴백 채널. 분쟁 증빙·긴 본문이 필요한 케이스.

| 코드 | 트리거 | 수신자 | 채널 | 비고 |
| --- | --- | --- | --- | --- |
| E01 | 환불 완료 | 멘티 | **알림톡 + 이메일** | 알림톡으로 즉시 확인, 이메일은 분쟁 증빙 보존용 |
| E02 | 멘토 승인/반려 | 멘토 | 이메일 전용 | 반려 사유는 본문 길이 제한 없는 이메일이 적합 |

### 제거된 템플릿 (이전 v1.1 14종 설계에서 빠짐)

> ⚠ 아래 항목들은 **재번호 이전**의 설계상 명세였음. 현재 운영되는 A01~A08은 위 표 참고. 코드명 충돌 방지를 위해 옛 코드는 표기하지 않음.

| 트리거 | 제거 사유 |
| --- | --- |
| 예약 신청 접수 (멘토에게) | 자동 확정 모델로 결제 완료 알림(A02)에 흡수 (멘토 수락 단계 자체 제거) |
| 멘토 수락 완료 (멘티에게) | 자동 확정 모델로 불필요 |
| 멘토 거절 (멘티에게) | 거절 기능 제거 (멘토가 직접 연 슬롯에 결제됐으므로 거절권 없음) |
| 사전질문지 미작성 리마인드 | 결제 시점에 답변 작성 강제로 변경 → 미작성 케이스 자체가 없음 |
| 환불 완료 (알림톡) | 이메일(E01)로 대체 |
| 멘토 승인/반려 (알림톡) | 이메일(E02)로 대체 |
| 클레임 접수/처리 결과 | 1:1 문의 시스템(docs/10 §23-1)으로 흡수 |

### 필수 요소

- 알림 확인 버튼 또는 확인 상태 기록 (A04, A05)
- 모든 발송 건 DB 로그 저장 (재발송 근거)
- 카카오 사전 심사 완료 템플릿만 운영 (영업일 기준 1~3일 리드타임)
- 거래성 알림과 마케팅성 알림 동의/채널 분리
- 마케팅 수신 미동의자에겐 거래성 알림만 발송
- 장애 발생 시 관리자 슬랙/이메일 실시간 경보
