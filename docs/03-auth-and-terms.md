# 03. 인증 및 약관

## 11-1. 회원가입 및 인증

### 가입 방식 (공통)

- 이메일/비밀번호 가입은 MVP에서 제공하지 않음
- **소셜 로그인 단일화**: 카카오, 네이버
- 동일 휴대폰 번호 기반 소셜 계정 병합 가능
- 세션 관리는 백엔드 기반 (JWT + Refresh 또는 세션 쿠키 택1)

### 가입 플로우 (순서 고정)

```
[1] 소셜 로그인 (카카오/네이버)
      ↓
[2] 역할 선택 (멘토 / 학생 / 학부모)
      ↓
[3] 약관 동의 (역할별 약관 구성)
      ↓
[4] 휴대폰 본인인증 (PASS 또는 NICE)
      ↓
[5] 역할별 추가 정보 입력
      ↓
[6] (멘토 한정) 증빙 서류 제출 → 관리자 검수 → 승인 후 프로필 노출
```

**순서 근거**

- 약관 동의는 본인인증(개인정보 수집 행위)보다 **반드시 앞**에 위치 — 개인정보보호법상 수집·이용 동의는 수집 전에 받아야 함
- 역할 선택이 약관 동의보다 앞에 있어야 **역할별 약관 세트**를 정확히 구성 가능 (멘토는 정산/과세/서비스 운영 약관 추가 필요)

### 멘티/학부모 기본 항목

- 이름
- 휴대폰 (본인인증)
- 역할 (멘티/학부모)

### 학부모-자녀 구조

- 학부모 계정 하위에 자녀(멘티) 프로필 **최대 5명**까지 등록 가능
- 예약/결제는 학부모 계정, 멘티 정보는 자녀 프로필 참조
- 만 14세 미만 멘티는 본인 가입 불가, 학부모 계정으로만 이용 가능
- 만 14세 미만 개인정보 수집 시 **법정대리인 동의 필수** (자녀 프로필 등록 시점)
- 자녀 프로필 추가 시 서버에서 현재 등록 수 체크, 5명 초과 시 422 반환

### 역할 겸용 정책

- 멘토가 **멘티 역할 겸용** 허용: 대학생 멘토도 다른 멘토(선배 등)에게 상담 받을 수 있음
- 단, 같은 사람이 **본인 상품**에 예약 불가 (서버에서 `booking.mentee_user_id === mentor.user_id` 차단)
- 역할별로 UI 분기: 멘토 대시보드와 멘티 마이페이지는 동일 계정에서 탭·메뉴로 전환

### 멘토 필수 항목

- 이름/닉네임
- 학교
- 단과대/학과
- 학년
- 입시 유형/경험
- 소개
- 상담 가능 주제
- 상담 가능 요일/시간
- 증빙 서류
- 정산 계좌 정보

### 멘토 검증 로직

- 재학/합격 증빙 제출
- 학교 이메일 인증 또는 대체 증빙
- 본인 인증
- 허위/과장 표현 검수
- 승인 전 프로필 비노출

- 필수 항목 **개별 동의 체크**가 원칙 (일괄 동의 버튼은 보조, 각 항목 재확인 가능해야 함)

- 만 14세 이상 확인(T03)은 **체크박스 + 생년 입력**으로 이중 확인

### 18-4. 결제 시 동의 (v1.2 — ADR-002로 클로즈)

- T13, T14는 결제 진행 시점에 노출 및 동의 수집
- **만 14세 이상 멘티 본인 결제**: **본인 동의만 수집** (T01~T04 + T13 + T14). 보호자 추가 동의 화면·통보 없음. 만 14세 이상은 약관규제법·정보통신망법상 본인 동의 능력 인정 (docs/11 ADR-002)
- **만 14세 미만 자녀**: 본인 결제 불가. 학부모 계정으로만 결제 진행(§학부모-자녀 구조 / §18-7 법정대리인 동의 절차). 변경 없음
- **학부모 결제 (자녀 명의)**: 자녀 만 14세 이상이라도 학부모 결제 자체는 허용. 이 경우 결제 시점 약관(T13·T14)은 학부모 본인이 동의

### 18-5. 약관 운영

- 약관 버전 관리 및 이력 보존 (user_id × 약관코드 × 버전 × 동의시각 DB 기록)
- 약관 개정 시 전체 사용자 재동의 플로우 (로그인 시 인터셉트)
- **재동의 화면은 가입 시 약관 동의 화면(`terms-{role}.html`)을 그대로 재사용. 별도 인터셉트 화면 신설 안 함** (운영·디자인 부담↓ · 1인 개발 환경 고려). 재동의 진입 시 쿼리 파라미터(`?reconsent=true&from_version=v1.x`)로 분기하여 변경 조항만 amber 톤 강조 + "개정된 약관에 다시 동의해 주세요" 헤더 카피 노출 가능 (구현 단계 결정)
- 선택 동의는 언제든 마이페이지에서 철회 가능
- 관리자 CMS에서 약관 본문/버전/시행일 관리

#### 재동의 grace period

필수 약관 개정 시 기존 사용자에 대한 재동의 강제 정책.

| 항목 | 값 |
|---|---|
| **grace period** | 시행일로부터 **14일** |
| **그레이스 기간 중 미동의 사용자** | 로그인은 허용. 신규 예약·결제·멘토 활동은 차단(재동의 모달 강제 노출) |
| **14일 경과 후 미동의** | **강제 로그아웃** + 다음 로그인 시 재동의 완료 전까지 일반 페이지 접근 차단 |
| **선택 약관(T10·T11·T12) 개정** | grace period 없이 마이페이지에서 자유 철회·재동의. 강제 인터셉트 없음 |
| **멘토 전용 약관(T05·T06·T07·T08) 개정** | 위와 동일 14일. §18-11 핵심 7종 조항 변경은 30일 사전 통지(T05-운영) 후 시행일부터 14일 grace |

**Why 14일**: 7일은 사용자 인지·검토 시간 부족(휴가·해외 부재 가능성), 30일은 핵심 약관 미동의 상태 누적이 길어 법적 리스크. 14일은 사용자 인지 보장 + 운영 부담 균형.

#### History
- 2026-04-28 · §18-5 재동의 grace period 14일 명시 — 사유: mockup `auth/terms-reconsent.html` 더미 카피("14일 후 자동 로그아웃") 정합 보강, T01·T02 핵심 약관 미동의 누적의 법적 리스크 차단
- 2026-04-28 · 재동의 전용 화면(`terms-reconsent.html`) 제거 명시 — 사유: 가입 시 동의 화면(`terms-{role}.html`)을 동일하게 재사용하며 쿼리 파라미터로 분기. 화면 중복 유지 부담↓ · 변경 조항 강조 표현은 구현 단계 결정. 동시 진행: terms.html을 역할별 3개(`terms-mentee/parent/mentor.html`)로 분리

### 18-6. 본인인증 예외

- 본인인증 대체 수단 MVP 미지원
- 14세 미만 보호자 가입 시 **보호자 본인인증**은 필수, 자녀 본인인증은 불요

### 18-7. 만 14세 미만 자녀 등록 UX (법정대리인 동의)

개인정보보호법상 만 14세 미만 개인정보 수집 시 법정대리인 동의 필수. UX 구현 규칙.

**트리거 시점**: 학부모 계정에서 "자녀 프로필 추가" 시, 자녀 생년 입력 결과 만 14세 미만으로 판정될 때.

**UX 플로우**:

1. 별도 모달로 전환 (일반 자녀 등록과 분기)
2. T09 약관 전문 노출 + 필수 체크
3. **보호자 본인인증 재확인** (이미 인증된 계정이라도 이 시점에 재확인 API 호출)
4. 자녀 정보 입력 (이름·학년·학교 등)
5. 최종 제출

**DB 기록 필수 항목**

- `user_id` (보호자)
- `child_profile_id`
- 동의 시각 (`agreed_at`)
- 동의 당시 IP
- 동의 당시 User-Agent
- 동의한 T09 버전

**운영 규칙**

- 보호자 본인인증이 만료/실패 상태면 자녀 등록 진행 불가
- 자녀가 만 14세가 되면 (생일 경과) 알림톡/이메일로 "본인 계정 전환 가능" 안내
- 동의 이력 파기 기한은 §18-8 참조

### 18-8. 개인정보 보관·파기 **정책**

**기본 원칙**: 탈퇴 즉시 개인정보 파기. 단, 법정 의무 보관 기간이 있는 데이터는 해당 기간 동안 **분리 보관** 후 파기.

| 데이터 유형                 | 보관 기간                            | 근거                     |
| --------------------------- | ------------------------------------ | ------------------------ |
| 계약·결제·청약철회 기록     | **5년**                              | 전자상거래법 §6          |
| 대금 결제 및 재화 공급 기록 | **5년**                              | 전자상거래법 §6          |
| 소비자 불만·분쟁 처리 기록  | **3년**                              | 전자상거래법 §6          |
| 세금 관련 정산 기록 (멘토)  | **5년**                              | 국세기본법 §85의3        |
| 로그인 기록·접속 IP         | **3개월**                            | 통신비밀보호법 §15의2    |
| 본인인증 기록               | **해당 서비스 종료 시까지**          | 정보통신망법             |
| 법정대리인 동의 이력        | **탈퇴 후 5년** (미성년자 보호 목적) | 회사 정책                |
| 후기 (익명화 후)            | **무기한**                           | 다른 멘티 선택 근거 보존 |

**파기 방식**

- DB: `DELETE` 또는 **anonymize** (이름·연락처·이메일 등 식별자를 hash 또는 `DELETED_USER_{id}`로 교체)
- 파일(S3): 영구 삭제
- 로그: 리텐션 정책에 따라 자동 삭제 (CloudWatch Logs retention)

**휴면 계정**

- 1년 이상 미접속 시 **휴면 전환** 및 개인정보 분리 보관 (별도 DB 테이블 또는 파티션)
- 휴면 전환 30일 전 이메일·알림톡 통지
- 사용자가 로그인하면 즉시 해제

### 18-9. 멘티·학부모 탈퇴 절차

**탈퇴 선행 조건**

- **예정된 예약이 있으면 탈퇴 불가** — 먼저 모두 취소(§13-1)하고 환불 완료 후 가능
- 진행 중인 클레임이 있으면 탈퇴 불가 (관리자 해결 대기)
- 결제 대기 중(`REQUESTED`) booking은 자동 `EXPIRED` 처리 후 탈퇴 가능

**탈퇴 처리**

1. 계정 `status = DELETED` 전환 (soft delete)
2. `users.name/nickname/phone` → `DELETED_USER_{user_id}` 로 익명화
3. `oauth_id`는 30일간 재가입 방지용으로 hash 보관 후 파기
4. 지난 예약·후기는 유지하되 작성자 표시는 "탈퇴한 사용자"로 노출
5. 법정 보관 대상 데이터(결제 등)는 §18-8 표대로 분리 보관

**재가입 제한**

- 동일 oauth_id로는 **탈퇴 후 30일 경과** 전 재가입 불가 (악용 방지)
- 30일 후 재가입 시에는 완전히 새 계정으로 처리 (지난 데이터 연결 안 함)

**학부모의 경우**

- 학부모 탈퇴 시 등록된 모든 자녀 프로필도 함께 익명화
- 자녀 명의 후기가 있다면 "탈퇴한 사용자"로 노출

### 18-10. 멘토 탈퇴 절차

**탈퇴 선행 조건**

- 예정된 예약(`REQUESTED`/`CONFIRMED`/`RESCHEDULE_PROPOSED`) **0건** 필수
- **미정산 금액 0원** — 직전 주간 정산까지 모두 수령한 상태 (주간 정산일 이후 탈퇴 권장)
- 진행 중 클레임·패널티 없음 (§20-5 기준)

**탈퇴 처리**

1. `mentor.status = DELETED` + 해당 user 계정도 §18-9 동일 프로세스
2. 상품은 모두 `status = DELETED` 전환, 탐색 미노출
3. 받은 후기는 **익명 멘토로 유지** (다른 사용자 참고 데이터 가치)
4. `rejection_code`/`mentor_payout_accounts` 등 정산·과세 정보는 §18-8 표대로 **5년 분리 보관**

**재가입 제한**

- 동일 oauth_id 재가입은 탈퇴 후 **90일 경과 후 가능** (멘티보다 긴 이유: 어뷰징 방지, 검수 부담)
- 재가입 시 멘토 승인 프로세스 처음부터 재개

### 18-11. 멘토-플랫폼 법적 관계 명시

별도 종이 계약서 없이 약관 동의만으로 정산 관계를 운영하기 위한 핵심 조항. **T05 멘토 서비스 운영 약관 본문에 반드시 포함**되어야 함.

#### 핵심 7종 조항 (T05~T08 분산 배치)

| 조항                    | 위치      | 내용                                                                                                                                      |
| ----------------------- | --------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **계약 성격 명시**      | T05       | 멘토와 플랫폼은 **위탁/프리랜서 관계**이며 근로계약이 아님. 4대보험·최저임금·퇴직금·연차 등 근로기준법상 근로자 보호 조항이 적용되지 않음 |
| **세무 처리 주체**      | T07       | 멘토 활동 소득에 대한 종합소득세 신고는 멘토 본인의 의무. 플랫폼은 원천징수만 수행하고 신고 대행 없음                                     |
| **정산 주기·차감 규칙** | T05 + T06 | 주간 정산 / 50% 수수료 / 패널티 비율 차감(§20-5) / 환불 시 멘토 정산 차감 가능에 사전 동의                                                |
| **취소·환불 동의**      | T05       | 멘티의 정당한 취소 요청 시 멘토 정산 영향이 발생함을 사전 인지·동의                                                                       |
| **저작권 귀속**         | T05       | 멘토가 등록한 사전질문·프로필 텍스트의 플랫폼 사용권 / 후기는 플랫폼 영구 사용권 / 상담 중 발화 내용은 멘토 귀속                          |
| **분쟁 해결**           | T01 + T05 | 준거법 한국법, 관할 법원 명시 (보통 회사 본점 소재지)                                                                                     |
| **약관 변경 시 통지**   | T05-운영  | 약관 개정 시 30일 전 통지 + 재동의 필요. 미동의 시 멘토 활동 일시 중단                                                                    |

#### 비독점성 명시

- 멘토는 다른 멘토링·교육 플랫폼·과외 등 외부 활동 **자유**
- 플랫폼은 멘토에게 전속 의무 부과 안 함
- 단 **외부 연락처·플랫폼 이탈 유도**(docs/11 §13-5)는 별개로 금지

#### 위탁 관계 유지 운영 원칙

근로자성 인정 위험을 낮추기 위한 **실질적** 운영 가이드.

- 멘토 가능시간은 본인 자율 설정 (강제 출근시간 없음)
- 멘토는 자신의 가격을 자율 책정 (관리자 설정 최저가~최고가 범위 내)
- 멘토링 플로우 템플릿(docs/12 §12-4)은 **권고**일 뿐 의무 아님
- 4대보험·복리후생 미제공 명시
- 보고 의무·정기 미팅 강제 없음

이 운영 원칙을 어기면 (예: 의무 출근, 고정급 지급, 강한 지휘감독) 위탁 관계가 부정되고 근로자로 분류될 위험이 있음 — 변경 시 노무사 자문 필수.

### 18-12. 전자상거래법 청약철회 적용 기준

전자상거래법 §17에 따른 청약철회 가능 여부.

| 시점 | 청약철회 가능 여부 | 근거 |
|---|---|---|
| 결제 완료 ~ 멘토링 시작 전 | 플랫폼 취소 정책(§13-1) 적용. 법정 7일 이내 전액 환불 기준은 취소 구간 정책이 더 유리하면 취소 정책 우선 | 전자상거래법 §17①, 소비자에게 불리한 경우 계약 조건 우선 |
| 멘토링 시작 후 | **청약철회 불가** — 결제 화면에서 "서비스 시작 시 청약철회 불가에 동의" 사전 고지 및 체크 후 진행 | 전자상거래법 §17②3 (용역 제공 시작 동의 시 예외) |
| PG 결제 오류·중복 결제 | 즉시 전액 환불 (회사 귀책) | 전자상거래법 §17①, 회사 귀책 |

**결제 화면 필수 고지 문구** (T13 §4에 포함):
> "멘토링이 시작되면 청약철회가 불가합니다. 결제 전 멘토 프로필 및 예약 시간을 반드시 확인해 주세요."

#### History
- 2026-05-04 · §18-12 신설 — 전자상거래법 청약철회 예외 기준 명시. T13 §4 본문과 연동. 결제 화면 필수 고지 문구 확정.

### 18-13. 통신판매중개업자 신고 (법무 자문 확인 필요)

아자스쿨 멘토링 서비스는 전자상거래법상 **통신판매중개업자**에 해당할 가능성이 높음.

| 항목 | 내용 |
|---|---|
| 해당 법령 | 전자상거래 등에서의 소비자보호에 관한 법률 §2⑨, §20 |
| 의무 | 공정거래위원회에 통신판매중개업자 신고, 면책 범위 공지 의무 |
| 면책 범위 | 통신판매중개업자는 통신판매 당사자(멘토)의 고의·과실로 인한 손해에 대해 면책 가능 (단, 자신에게 귀책이 없는 경우) |
| T01 반영 | "회사는 전자상거래법상 통신판매중개업자로서 멘토와 멘티 사이의 거래를 중개합니다" 문구 포함 |
| **런칭 전 필수** | 변호사 확인 + 공정거래위원회 신고 (신고번호를 T13 사업자 정보란에 기재) |

#### History
- 2026-05-04 · §18-13 신설 — 통신판매중개업자 신고 요건 정리. 법무 자문 체크리스트(§13-7-1)에 추가. T01 §4, T13 §1 사업자 정보에 신고번호 기재 의무 연동.
