## 13. 정책

### 13-1. 취소/환불 정책

#### 멘티 측 취소
| 시점 | 환불 |
|---|---|
| 24시간 이전 | 전액 환불 |
| 3~24시간 전 | 50% 환불 |
| 3시간 미만 | 환불 없음 |
| 멘티 노쇼 | 환불 없음 |

#### History
- 2026-04-28 · 3구간 유지 확정 (중간 구간 6h·12h 추가 안 함) — docs/11 ADR-002로 §25-2 클로즈. MVP 환불 계산식 단순화 우선

#### 멘토 측 취소 (상세는 [docs/05 §20-5](./05-booking-and-sessions.md#20-5-멘토-일정-변경취소노쇼-정책))
- 멘티에게는 시점 무관 **전액 환불**
- 멘토에게는 누적 경고 기반 비율 패널티 (5% → 10% → 20% → 40% → 80%, 곱으로 증가)
- 3회 누적 시 1개월 프로필 정지, 5회 누적 시 계정 정지
- 상담 시작 72h 이전까지만 "변경 제안" 가능, 이후는 취소만

#### 품질 클레임
- 관리자 검토 후 처리 (처리 단계는 §13-2 참고)

### 13-2. 클레임 처리 기준
- 시간 미준수
- 무성의한 진행
- 허위 이력 의심
- 부적절 언행
- 진행 방식 불일치

### 13-3. 노출 정책
- 승인 전 멘토 비노출
- 후기 조작/허위 정보 적발 시 노출 중지 가능
- 인기 멘토 영역은 운영 기준에 따라 관리

## 13-5. 외부 연락처·광고 금지 정책 (T2-11)

플랫폼 이탈 유도·사기·개인정보 유출 방지.

**금지 노출 영역**
- 멘토 프로필 (이름·소개·키워드·학교란 등)
- 멘토링 상품 (제목·설명·준비사항·기대효과)
- 후기·답글
- 상담 중 채팅 (녹화 동의된 경우 포함)

**금지 항목**
- 전화번호 (010-XXXX-XXXX, +82-10 등 패턴)
- 카카오톡 ID, 카카오 오픈채팅 링크
- 인스타그램 · 페이스북 · X · 유튜브 핸들 또는 URL
- 이메일 주소
- 블로그·웹사이트 URL (아자스쿨 도메인 제외)
- 외부 결제/송금 유도 문구 ("계좌이체로 직접 결제", "수수료 없이 직거래" 등)

**탐지 방식**
- 저장 시 정규식 기반 자동 탐지 (전화번호·이메일·URL 패턴)
- 저장 거부 + 에러 메시지 노출 ("외부 연락처는 프로필에 등록할 수 없습니다")
- 자동 탐지 회피(한글 표기, 띄어쓰기 등)는 신고 기반 수동 처리

**제재 단계**
- 1차 적발: 경고 + 해당 문구 강제 삭제
- 2차 적발: 7일간 신규 상품 등록 제한
- 3차 적발: **계정 정지 30일 + 해당 기간 정산 보류**
- 명백한 사기 의도 시: 즉시 영구 정지 + 수사기관 신고 가능

## 30. 정책 결정 기록 (ADR)

큰 정책 변경은 ADR(Architecture Decision Record) 형식으로 누적 기록한다. 채택·기각 사유, 영향 범위를 한 곳에 모아 두어 추후 재논의 시 컨텍스트를 잃지 않게 한다.

### ADR-001 · 노쇼 자동 판정 + 자동 환불 도입 (2026-04-28)

**상태**: Accepted

**컨텍스트**
- 기존 §20-3은 "시작 +10분 경과 + 상대방 미입장 시 신고 버튼 활성, 미신고 시 자동 NO_SHOW 확정" 수준만 정의되어 있었다. 자동 판정 시점·환불 트리거·관리자 큐 동선은 미정의.
- 1인 풀스택 운영 체제(Owner: dev1@ajaschool.co.kr)에서 멘티가 신고를 누락한 채 이탈하면 환불이 누락되고, 멘토 패널티 카운트도 적재되지 않아 운영 부담과 신뢰도 리스크가 동시에 발생.
- 양측 미입장 케이스(advisor 보고서 §4-3 운영흐름 모호 항목)에 대한 처리 절차도 명시되지 않아 클레임 발생 시 일관 처리 불가.
- `specs/feature-spec.md` 작성 중 §6.1.4 (시스템·횡단)와 §2.4.3·§4.5.5·§5.4.1 사이의 시간 일관성을 잡으려다 정책 갭이 발견됨.

**결정**
- 상담 시작 +**15분** 시점에 시스템 cron(ShedLock 기반 단일 인스턴스 보장)이 `session_entry_log`를 검사하여 입장 row가 0인 측을 자동으로 노쇼 확정한다.
- 멘토 노쇼 또는 양측 노쇼 시 **즉시 100% 환불**(토스페이먼츠 환불 API 자동 호출). 멘티 노쇼는 §13-1대로 환불 없음.
- 자동 처리 결과는 관리자 슬랙 + 관리자 대시보드 "노쇼 자동 처리 큐"로 **사후 알림**. 운영자는 이의·롤백을 사후 검토할 수 있다.
- 멘토 패널티 카운트 §20-5는 노쇼 가중치 **+2** 그대로 유지. 멘티 §20-8 가중치 **+1**도 유지.
- 멘티 신고 버튼은 폴백으로 유지하되 활성 시점을 +10분 → +**15분** 이전까지로 통일. 신고와 자동 판정의 결과(상태·환불·패널티)는 동일하다.
- 양측 미입장은 별도 "양측 노쇼" 큐로 분류 — 100% 환불 + 양측 패널티 카운트 적용 + 관리자 사후 점검.

**대안**
| 대안 | 평가 | 채택 여부 |
|---|---|---|
| 신고 기반만 유지 | 환불 누락 리스크 + 운영 부담. 사용자가 +15분간 대기 후 이탈하면 추적 불가 | 기각 |
| 자동 판정 후 관리자 사전 승인 후 환불 | 환불 지연(영업일 1일 이상) + 1인 운영자 부담 | 기각 |
| **자동 판정 + 즉시 환불 + 관리자 사후 검토(이의 시 롤백)** | 환불 즉시성 + 운영 부담 최소 + 이의 통로 확보 | **채택** |

### ADR-002 · MVP 미결 정책 5건 일괄 클로즈 (2026-04-28)

**상태**: Accepted

**컨텍스트**
- `specs/feature-spec.md` 작성 중 §25 미결 5건이 명세서 옵션 컬럼·MVP 분류·ERD에 동시 영향. 1인 풀스택 + 4주 MVP 부담 최소화 + 중복 ADR 추적 부하 회피 관점에서 5건을 동시에 클로즈할 필요 발생.
- 각 항목별 ADR 분리는 시스템 도입 수준이 아니므로(쿠폰 자동화·결제대행 신설 모두 OUT-W5) 단일 ADR로 묶어 운영 부담을 줄이고 §25 표를 한 번에 비운다.

**결정**

1. **취소·환불 시간 구간(25-2)** — 현재 3구간(24h↑ 100% / 3~24h 50% / 3h↓ 0%) 유지. 중간 구간(6h·12h) 추가하지 않음. 신규 분기 없이 멘티 의사결정 부담을 단순화하고 환불 계산 로직 1단순화 우선.
2. **쿠폰·프로모션 정책(25-7)** — **발급은 OUT-W5(운영자 수동 SQL/엑셀, 클레임 보상 한정)**, **사용 UI만 MVP IN-수동**(`§5.2.x` 발급 동선 포함). 결제 화면에서 코드 입력 → 할인 적용 → 결제 금액 차감. 자동 발급·자동 만료 알림·코드 자동 지급 모두 미구현. 정산은 멘토 기준가 기준 50:50 그대로 유지(쿠폰 할인은 플랫폼 부담).
3. **가격 권장 상한(25-12)** — **Advisory only**. 멘토 상품 등록 시 권장 상한(₩99,900) 표기·경고만 표시, 시스템 차단 없음. floor(₩29,900) + 신규 멘토 5건 락은 그대로 유지. 시장 반응 보고 W5+에서 상한 강제 여부 재검토.
4. **14~17세 멘티 본인 결제 시 보호자 추가 동의(18-4)** — **본인 동의만 수집** (T01~T04 + T13 + T14). 보호자 추가 동의 화면·통보 없음. 만 14세 미만은 기존대로 학부모 강제 결제 정책 유지(변경 없음). 이유: 만 14세 이상은 약관규제법·정보통신망법상 본인 동의 능력 인정. MVP에서 이중 동의 분기 신설은 1인 운영 부담만 가중.
5. **payer/user 분리 데이터 모델(ERD)** — `bookings` 테이블에 `user_id`(이용자) + `payer_id`(결제자) **별도 컬럼**. 일반 케이스는 두 값 동일(본인 결제), 학부모 케이스는 분리(payer = 학부모, user = 자녀). 향후 결제대행 시나리오까지 동일 컬럼으로 커버. `payments` 테이블은 토스 transaction용으로 별도 유지하며 `booking_id` FK + `payer_id` 참조.

**대안**

| 25-2 | 6h·12h 중간 구간 추가 (예: 12h↑ 75%) | 환불 계산식 분기 증가 + 멘티 의사결정 모호 + MVP 트랜잭션 단순화 우선 |

## ADR-003: 가격 모델 전환 — 강제 floor 폐지 + 관리자 설정 범위 자율

**날짜**: 2026-05-04

**상태**: Accepted

**컨텍스트**
- 기존 강제 floor(₩29,900)는 멘토 온보딩 마찰 요인. 의대·SKY 멘토가 "5건 채울 때까지 3만원만 받아야 함"을 부담으로 느껴 등록 포기 가능성 존재.
- 대표 미팅에서 "최저가·최고가 모두 운영팀이 시장 반응 보며 조정하는 게 맞다"는 결론 도달. 29,900원은 시장 기준선으로만 활용.

**결정**
1. **강제 floor 폐지** — `mentoring_products.price >= 29900` DB CHECK 제거. 서비스 레이어 검증을 `platform_settings.price_min` 기준으로 전환
2. **관리자 설정 범위 도입** — `platform_settings` 테이블에 `price_min`·`price_max` 컬럼 추가. 런칭 전 대표가 초기값 확정
3. **29,900원 권장가 유지** — UI 가격 input의 placeholder/기본값으로 "권장 ₩29,900" 레이블 표시. 강제 아님
4. **신규 멘토 floor 락 폐지** — 관리자 `price_min`이 품질 하한을 보호하므로 별도 락 불필요

**대안**

| 기각된 대안 | 사유 |
|---|---|
| floor 금액만 낮춤 (예: ₩10,000) | 운영팀이 시장 반응 보며 동적으로 조정하려면 코드 배포 없이 가능해야 함 → admin 설정 방식이 우월 |
| 신규 멘토 락만 유지 | floor가 없으면 락 기준 자체가 모호해짐 → 일관성 위해 함께 폐지 |
| 상한만 admin 설정, 하한은 hardcode | 비대칭 설계. 운영 유연성 측면에서 양쪽 모두 admin 설정이 일관됨 |

### ADR-004 · 미성년자 결제 민법 §5 리스크 보완 (2026-05-04)

**상태**: Accepted

**컨텍스트**
- ADR-002에서 만 14세 이상 멘티 본인 결제를 허용했으나, 이는 개인정보보호법상 동의 능력 기준이다.
- 민법 §5에 따르면 만 19세 미만 미성년자의 법률행위는 법정대리인의 동의 없으면 취소 가능하다.
- 고등학생(만 15~18세)이 1차 타겟인 서비스에서 이 리스크를 무시하면 분쟁 발생 시 전액 환불 의무 + 운영 신뢰도 훼손이 우려된다.
- ADR-002 결정(보호자 추가 동의 화면 신설 안 함)은 유지하되, 고지 의무와 결제 화면 확인 절차를 추가한다.

**결정**
1. **T01 + T13에 민법 §5 고지 조항 명문화** — "만 19세 미만 이용자는 법정대리인 동의 없이 체결된 계약은 민법에 따라 취소될 수 있음" 명시 (docs/19-terms-content.md T01 §이용계약 체결, T13 §5항)
2. **결제 화면 미성년자 확인 UI (W2 Day 6~7)** — 가입 시 생년 입력 기반으로, 만 19세 미만 이용자가 결제 진행 시 "법정대리인(부모·보호자)의 동의를 받고 결제합니다" 체크박스 1개 추가. 필수 체크.
3. **만 19세 미만 결제 기록 강화** — bookings 테이블에 `is_minor_confirmed BOOLEAN DEFAULT FALSE` 컬럼 추가. 체크박스 체크 시 TRUE 기록. 분쟁 시 증거 활용.
4. **고지 우선 전략 유지** — ADR-002처럼 별도 보호자 동의 화면 신설은 안 함. 고지 + 확인으로 분쟁 예방.
