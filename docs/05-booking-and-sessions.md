# 05. 예약 및 상담

## 11-4. 예약/일정 조율

### 기능
- 멘토 가능 시간 등록 (7일 이후 슬롯만)
- 멘티가 슬롯 선택 후 결제
- **결제 완료 시 자동 예약 확정** (멘토 수락 단계 없음)
- 멘토는 확정 후 **변경 제안**만 가능 (취소는 패널티 §20-5)

### 원칙
- 멘토가 직접 연 슬롯이므로 **거절권 없음** — 결제되면 무조건 진행
- 신변상 사유 있을 시 변경 제안 또는 취소(패널티)로 우회
- 자동 확정 모델로 운영 단순화·결제 전환율 최대화

### 멘토 가능시간 등록 리드타임

멘토가 가능시간을 성급하게 열었다가 번복하는 사고 방지용 제약.

| 작업 | 제약 |
|---|---|
| 가능시간 슬롯 **추가** | 오늘로부터 **7일 이후** 날짜만 허용 |
| 가능시간 슬롯 **삭제** | 해당 슬롯 시작 시각이 **14일 이상 남은 경우**만 자유 삭제 |
| 휴무 블록 추가 (`time_block BLOCKED`) | 상담 시작 **14일 이상 남은 경우**만. 그 안쪽은 "일정 변경 제안" 플로우로 우회 |

해당 기간 안쪽에 이미 예약된 건이 있으면, 삭제가 아니라 **§20-5 멘토 일정 변경 제안** 플로우를 타야 한다.

### 멘토 자발적 일시정지 (휴가)

- `mentor.status = ON_LEAVE` 전환 (신규 상태값)
- 최대 연속 **30일**까지 설정 가능. 만료 시 자동 `APPROVED` 복귀 스케줄러
- 휴가 중에는 **탐색·검색 미노출**, 가능시간 슬롯 자동 숨김
- **기존 확정 예약(`CONFIRMED`)은 그대로 진행** — 휴가를 이유로 자동 취소 안 됨
- 휴가 시작일로부터 **14일 내에 잡히는 신규 예약은 차단** (갑작스런 잠적 방지)
- 휴가 중 클레임·관리자 문의는 정상 접수 가능

## 11-7. 상담 진행

### 방식
- Zoom, Google Meet 등 외부 툴 사용
- **MVP**: 멘토가 직접 회의 생성 후 링크를 플랫폼에 등록
- **P1 (Week 5+)**: 멘토 개인 Zoom 계정 OAuth 연동 → 예약 확정 시 해당 멘토 계정에 미팅 자동 생성, 양측에 링크 자동 발송 (멘토 수동 등록 단계 제거)
- **P2 (장기)**: 플랫폼 Zoom 조직 계정에서 미팅 생성 또는 자체 화상 시스템 검토 — 라이선스 비용/동시 미팅 제한으로 MVP·P1 제외

### Zoom 연동 단계별 로드맵

MVP에서는 Zoom API 자동 생성을 제외하고 멘토 직접 링크 입력으로 운영한다.
고도화 단계별 비교:

| 방식 | 설명 | 유료 계정 필요성 | 장점 | 단점 | 추천 단계 |
|---|---|---|---|---|---|
| **멘토 직접 링크 입력** | 멘토가 Zoom/Meet 생성 후 플랫폼에 입력 | 낮음 (Zoom Basic 무료로 30분 운영 가능) | 개발 빠름, 비용 낮음, API 연동 불필요 | 링크 누락·오입력 가능, 플랫폼 통제력 낮음 | **MVP** |
| **멘토 Zoom OAuth 연동** | 멘토 계정으로 OAuth 연동 후 API 자동 생성 | 중간 (무료 계정도 가능, 안정 운영은 유료 권장) | 자동화 가능, 각 멘토 계정이 호스트라 동시 미팅 병목 없음 | OAuth/토큰 만료 대응 필요, 무료 계정 40분 제한 리스크 | **P1** |
| **플랫폼 Zoom 조직 계정** | 아자스쿨 계정에서 모든 미팅 생성 | 높음 (Business 이상, 동시 미팅·라이선스 비용 큼) | 플랫폼 통제력 최고 | 동시 미팅 제한, alternative host는 같은 조직 Licensed 사용자만 가능 | **P2** |
| **자체 화상 시스템** | WebRTC 등으로 직접 구축 | 매우 높음 (개발·운영비) | 완전한 통제 | 개발·운영 비용 큼 | **장기 검토** |

#### 기술 제약 사항

- **Zoom Basic 무료 계정**: 거의 모든 미팅에 40분 제한 → 30분 멘토링은 기술적으로 가능하나 시간 초과 리스크 존재
- **Zoom API 인증**: JWT 방식 폐기됨. Server-to-Server OAuth(플랫폼 자체 계정용) 또는 OAuth(멘토 외부 계정 연동용)로 구분
- **Server-to-Server OAuth**: 플랫폼 계정 단위 내부 앱. 동시 미팅은 계정 플랜 제한에 종속 (Pro: 동시 1개)
- **Alternative host 조건**: 같은 Zoom 계정 조직 내 Licensed 사용자만 가능 → 외부 멘토를 대체 호스트로 지정하려면 조직 편입 필요
- **P1 권장 구조**: 멘토가 개인 Zoom 계정을 OAuth로 플랫폼에 연동 → 예약 확정 시 해당 멘토 계정에 미팅 생성 → 플랫폼 단일 계정의 동시 미팅 병목 없음

### 상담 링크 정책
- 링크는 멘토가 직접 Zoom 또는 Google Meet에서 생성
- **허용 도메인**: `zoom.us`, `meet.google.com` (그 외 툴은 MVP 미지원)
- **예약 건별로 새 링크 등록 필수** (개인 상설 룸 재사용 금지 — 이전 참가자 혼입 방지)
- 등록 시 URL 형식 검증 (도메인 화이트리스트 + 경로 패턴)
- 멘토는 등록 후 상담 시작 15분 전까지 수정 가능, 이후 수정 불가

### 링크 공유 방식
- 플랫폼은 실제 URL을 알림톡/SMS에 직접 노출하지 않음
- 대신 마이페이지 내 **"상담 입장" 버튼**을 통해서만 접근 (링크 유출 방지 + 플랫폼 체류 유도)
- "상담 입장" 버튼은 상담 시작 **15분 전**부터 활성화
- 버튼 클릭 시 외부 Zoom/Meet URL로 새 탭 이동 및 **"입장 기록" 자동 저장**

### 운영 요구
- 예약 확정 시 사전질문지 작성 안내 발송 (A02)
- 멘토 링크 등록 완료 시 멘티에게 "상담 입장 준비 완료" 알림 발송 (A03)
- 상담 전 사전 질문을 멘토에게 전달
- 상담 종료 후 완료 처리 필요
- 상담 종료 후 상담 링크는 마이페이지에서 비활성화 처리 (재방문 차단)

### 상담 녹화 · 녹음 정책

- **원칙 금지**. 플랫폼은 녹화·녹음 기능을 제공하지 않음
- 양측(멘토·멘티)의 **사전 명시적 동의**가 있을 때만 녹화 허용
- 상담 시작 전 구두로 동의 확인 + 녹화 파일 사용 목적·보관 기간 상호 합의 의무
- **14세 미만 멘티는 법정대리인 동의 추가 필수**
- 위반 적발 시 클레임 대상 — 계정 경고 → 반복 시 정지
- 상담 링크 등록 시 UI 상단에 안내 배너 노출: "녹화·녹음은 양측 동의 하에만 허용"
- T05(멘토 서비스 운영 약관)·T01(이용약관)에 해당 조항 명시

## 20. 예약 상태 전이 상세 (v1.1)

### 20-1. 상태 정의

| 상태 | 의미 |
|---|---|
| `REQUESTED` | 결제 진행 중 (30분 내 완료 필요, 슬롯 락 유지) |
| `CONFIRMED` | 결제 완료 (자동 확정, 멘토 수락 단계 없음) |
| `RESCHEDULE_PROPOSED` | 멘토 또는 멘티의 일정 변경 제안 대기 |
| `EXPIRED` | 결제 30분 내 미완료로 자동 만료 |
| `CANCELED` | 멘티 취소 (환불 규정 적용) |
| `CANCELED_BY_MENTOR` | 멘토 취소 (패널티 대상) |
| `COMPLETED` | 상담 정상 종료 |
| `NO_SHOW_MENTEE` | 멘티 노쇼 |
| `NO_SHOW_MENTOR` | 멘토 노쇼 |

> `REJECTED` 상태는 자동 확정 모델 도입으로 제거됨. 멘토는 확정 후 변경 제안 또는 취소(패널티)만 가능.

### 20-2. 일정 변경 규칙 (개요)

세부 정책은 §20-5 (멘토 측), §20-6 (사유 코드), §20-7 (긴급 면제) 참조.

- 멘토 변경 제안: **상담 시작 72h(3일) 이전까지만** 허용, 멘티 승인 필요, 건당 **1회** 제한
- 멘토 대체 슬롯 **3개 제시** 의무 (멘티가 택1 or 전체 거절)
- 멘티 변경 요청: 결제 완료 이전 자유, 이후엔 멘토 승인 필요, 건당 **1회** 제한
- 멘티 응답 시한: **24h 내 미응답 시 자동 유지** (원 일정으로 확정)
- 멘티가 변경 제안 거절 시 → 원 일정 유지 or 멘토 취소 선택 (취소 시 §20-5 패널티 적용)

### 20-3. 노쇼 판정

자동 판정과 사용자 신고 두 분기가 병행한다. 두 경로 모두 결과(상태·환불·패널티)는 동일하며, 신고는 자동 판정 이전(시작 +15분 이전)의 폴백 경로 역할을 한다. 자동 판정 도입 사유와 대안 비교는 [docs/11 ADR-001](./11-policies-and-decisions.md#adr-001--노쇼-자동-판정--자동-환불-도입-2026-04-28) 참고.

#### 공통: 입장 기록 정의 (`SessionEntry`)

- **정의**: 마이페이지의 "상담 입장" 버튼을 클릭한 시각을 BE가 기록한 로그(`session_entry_log`).
- **기록 시점**: 버튼 클릭 → 외부 Zoom/Meet URL로 새 탭 이동 직전. 멘토·멘티 각각 별도 row.
- **활성 윈도우**: 시작 −15분 ~ 종료 +30분 사이 클릭만 유효 입장으로 간주.
- **노쇼 판정 근거**: 해당 booking 기준 입장 row가 `0`인 측을 "미입장"으로 본다.

#### 20-3-1. 자동 판정 분기 (시스템 cron)

- **트리거**: 상담 시작 시각 + **15분** 시점에 시스템 스케줄러(ShedLock 기반)가 검사.
- **판정 분기**:
  | 입장 기록 상태 | 확정 상태 | 환불 | 패널티 |
  |---|---|---|---|
  | 멘토 입장 0 / 멘티 입장 ≥1 | `NO_SHOW_MENTOR` | 100% 자동 환불 (토스페이먼츠 API) | 멘토 §20-5 카운트 **+2** |
  | 멘티 입장 0 / 멘토 입장 ≥1 | `NO_SHOW_MENTEE` | 환불 없음 | 멘티 §20-8 카운트 **+1** |
  | 양측 입장 0 | 양측 노쇼 — 멘티에게 100% 환불 + 양측 패널티 카운트 (멘티 §20-8 +1, 멘토 §20-5 +2) | 100% 환불 | 양측 카운트 모두 적용 |
- **부수 작용**:
  - 자동 환불 결과(성공·실패)는 관리자 슬랙 채널 + 관리자 대시보드 "노쇼 자동 처리 큐"로 사후 알림(이의·롤백 검토 가능).
  - 양측 노쇼는 별도 "양측 미입장" 큐로 분류하여 운영자가 진위·이중 알리바이를 사후 점검.
  - 자동 환불 API 호출 실패 시 booking 상태는 확정하되 환불은 `refund.status = FAILED`로 남겨 관리자 수동 재처리 큐에 적재.
- **사유 코드**: `noshow_detection_source = AUTO` 로 기록 (docs/14 §4-5 참조).

#### 20-3-2. 사용자 신고 분기 (폴백)

- **활성 시점**: 상담 시작 시각 + **15분 이전**까지 멘티(멘토 노쇼 시) 또는 멘토(멘티 노쇼 시)가 마이페이지에서 "노쇼 신고" 버튼으로 즉시 접수 가능. +15분 이후는 §20-3-1 자동 판정이 우선이므로 신고 버튼은 비활성.
- **목적**: 자동 판정을 기다리지 않고도 멘티가 빨리 환불 트리거를 잡을 수 있게 하는 폴백.
- **신고 진입 위치**: 멘티는 마이페이지 홈(`mockups/mypage/home.html`) "다음 상담" 카드, 멘토는 멘토 대시보드(`mockups/mentor/dashboard.html`) "예정된 상담" 카드의 각 상담 카드 우상단. (예약 신청 단계 화면인 `schedule.html`·`booking/mentor-detail.html`은 부적합 — 확정된 상담 카드에만 노출)
- **결과**: 자동 판정과 동일(상태·환불·패널티 결과 동일). `noshow_detection_source = REPORTED`.
- **이의**: 신고 후 자동 판정 결과와 충돌(상대가 +14분에 입장 등) 시 관리자 검토 큐로 자동 이관.

#### 20-3-3. 시간 충돌 정리 (v1.2)

이전 버전(v1.1)은 신고 버튼 활성 시점을 "시작 +10분"으로 명시했다. v1.2부터 자동 판정 도입에 맞춰 **+15분으로 통일** (자동·신고 동시 트리거). §20-5·§20-8의 "+10분" 표현도 모두 **+15분**으로 동기화.

#### 20-3-4. 관리자 검토 트리거

자동 판정 또는 신고 후, 다음 조건이면 관리자 검토 큐로 자동 이관:
- 양측 입장 0 (양측 노쇼 분류)
- 자동 판정 ↔ 신고 결과 충돌 (예: 멘티 +14분 신고 후 멘토가 +14분 30초에 입장 row 1건 적재)
- 환불 API 실패
- 멘토·멘티가 §20-7 긴급 면제를 첨부한 경우

#### 20-3-5. 자동 판정 이의 처리

자동 판정 결과에 대한 멘토·멘티의 이의는 **별도 윈도우 없이 1:1 문의(§23-1)로 단일 처리**. 운영자는 사실 확인 후 잘못된 판정이면 `noshow_detection_source = ADMIN_OVERRIDE`로 수동 변경(§14 §4-5 참조). 환불·패널티 정정도 같은 동선으로 사후 처리.

**Why:** MVP는 별도 이의 채널 신설 부담을 줄이고 1:1 문의 큐로 통합. 1인 운영 환경에서 큐 단일화가 실수·누락 방지에 더 안전.

### 20-4. 상담 링크 운영
- 멘토는 예약 확정 시점부터 상담 시작 2시간 전까지 링크 등록 필수
- 등록 시 URL 형식 검증: 허용 도메인 `zoom.us`, `meet.google.com`만 통과
- 예약 건별 신규 링크 등록 원칙 (개인 상설 룸 재사용 금지)
- 미등록 시 멘토에게 리마인드 (확정 후 24h, 시작 2h 전)
- 당일 2h 전 미등록 시 관리자 알림 에스컬레이션
- 멘토 수정 가능 구간: 등록 시점 ~ 상담 시작 15분 전

### 링크 노출/공유 원칙
- 알림톡·SMS에는 URL 직접 노출 금지 (마이페이지 진입 링크만 발송)
- 멘티 "상담 입장" 버튼은 시작 15분 전 활성화 → 외부 URL로 이동
- 버튼 클릭 시각을 입장 기록으로 저장 (노쇼 판정 근거)
- 상담 종료 처리 후 마이페이지 상담 입장 버튼 비활성화

### 20-5. 멘토 일정 변경·취소·노쇼 정책

멘티 쪽 정책(13-1)은 이미 확립되어 있고, 멘토 쪽 신뢰도 보장을 위한 제재 체계.

#### 시점별 허용 행위

| 상담 시작 기준 | 허용 행위 |
|---|---|
| ~ 72h 이전 | **변경 제안** 또는 **취소** 모두 가능 |
| 72h ~ 24h 전 | **변경 제안 불가**. 증빙 첨부한 **취소만** 가능 (패널티 적용) |
| 24h 미만 | **변경 제안 불가**. 취소 시 당일 노쇼 수준 패널티 |
| 상담 시작 후 15분 경과 + 미입장 | **노쇼 판정 대상** (§20-3·자동) |

#### 누적 경고 카운트 기반 패널티 비율

신뢰도 훼손이 누적될수록 제재가 **곱으로** 강화.

| 누적 경고 카운트 | 해당 건 정산 차감 비율 | 추가 조치 |
|---|---|---|
| 1회 | 5% | — |
| 2회 | 10% | — |
| 3회 | 20% | **1개월 프로필 정지** |
| 4회 | 40% | (정지 복귀 후) |
| 5회 | 80% | **계정 정지** |

6개월 롤링 윈도우. 6개월 이전의 경고는 카운트에서 제외.

#### 시점별 경고 카운트 가중치

| 행위 | 카운트 증가 | 비율 페널티 적용 |
|---|---|---|
| 72h 이전 취소 | +1 | 첫 경고는 기록만 (비율은 이후 발생 시부터) |
| 72h ~ 24h 전 취소 | +1 | 현재 카운트 기준 즉시 적용 |
| 24h 미만 취소 | **+2** | 현재 카운트 기준 즉시 적용 |
| 노쇼 (§20-3 확정) | **+2** | 해당 건 **정산 0%** (100% 차감) + 2주간 탐색 노출 하향 |

#### 멘티 보상 (모든 멘토 유책 케이스 공통)
- **상담료 전액 환불**
- 쿠폰 1매 보상 지급 (런칭 이후 쿠폰 시스템 도입 후 적용, 그 전엔 수동 사과 보상)

#### 누적 제재 예시

| 시나리오 | 결과 |
|---|---|
| 신규 멘토가 72h 이전 취소 | 카운트 0→1, 비율 0% |
| 같은 멘토가 다음에 24h 미만 취소 | 카운트 1→3, **20% 차감 + 1개월 정지** |
| 정지 복귀 후 노쇼 | 카운트 3→5, **80% 차감 + 계정 정지** |

### 20-6. 변경·취소 사유 코드 (드롭다운)

통계·패턴 탐지·악용 방지용. 변경 제안·취소·노쇼 모든 케이스에 **필수 선택**.

| 코드 | 사유 |
|---|---|
| `HEALTH` | 건강 문제 |
| `ACADEMIC` | 학업(시험·수업 충돌) |
| `FAMILY` | 가족 사정 |
| `SCHEDULE_CONFLICT` | 일정 중복 실수 |
| `OTHER` | 기타 (자유 입력 필수) |

`OTHER` 선택 시 **100자 이상 상세 사유** 입력 강제. 빈 사유로 취소 불가.

### 20-7. 긴급 사유 면제 (어뷰징 방지)

건강·가족 등 실제 불가항력 상황에 대한 패널티 면제 통로.

- 마이페이지에 "긴급 사유 신고" 버튼 → 증빙 첨부 업로드 (진단서·사고 증명 등)
- **관리자 검토 후 승인** 시 해당 건 패널티 및 카운트 **면제**
- **연 최대 2회**까지 면제 가능
- 2회 초과 신청 시 일반 취소 패널티 그대로 적용
- 허위 증빙 적발 시 **즉시 계정 정지 + 기존 면제분 소급 패널티 부과**

### 20-8. 멘티 노쇼 제재 체계

멘티 쪽 대칭성 보장. 환불 차단만으로는 반복 어뷰징·고의적 시간 낭비 방지 부족.

#### 판정 기준
- 상담 시작 **15분 경과 + 멘티 미입장**이면서 멘토는 정상 입장 (자동 판정 — §20-3-1)
- 멘토는 시작 +15분 이전까지 폴백 신고 가능 (§20-3-2). 결과(환불 없음·카운트 +1)는 자동 판정과 동일
- 양측 입장 0이면 멘티도 §20-8 카운트 +1 (양측 노쇼 분류)

#### 누적 제재 (6개월 롤링)

| 누적 노쇼 | 제재 |
|---|---|
| 1회 | 경고 알림톡 발송 (기록만) |
| 2회 | 경고 알림톡 + 다음 예약 전 "노쇼 방지 안내" 모달 강제 열람 |
| 3회 | **신규 예약 24시간 쿨다운** (3회째 노쇼 시점부터 24h간 예약 버튼 비활성) |
| 4회 | **신규 예약 72시간 쿨다운** + 계정 경고 배지 |
| 5회 | **계정 정지 30일** (자동 해제), 관리자 수동 영구정지 전환 가능 |

#### 환불 정책 (기존 13-1 그대로)
- 멘티 노쇼 시 환불 없음
- 멘토는 상담료 전액 수령 (정상 완료와 동일 처리)

#### 긴급 사유 면제
- §20-7과 동일한 통로 제공 (연 2회)
- 보호자가 대신 신고 가능 (미성년자 멘티 케이스)
