# 01. 제품 정의

## 1. 제품 개요

### 1-1. 제품명
아자스쿨 온라인 멘토링

### 1-2. 한 줄 정의
중고등학생과 학부모가 대학생 멘토에게 진학, 공부법, 전공 탐색, 학교생활, 입시 경험 관련 상담을 온라인으로 신청하고 진행하는 멘토링 중개 서비스

### 1-3. 제품 포지션
- 패키지형 캠퍼스투어/오프라인 체험 판매 플랫폼이 아님
- 대학생 공강 시간을 즉시 판매하는 오픈마켓이 아님
- 학생/학부모의 고민을 해결해 줄 검증된 대학생 멘토를 연결하는 온라인 멘토링 서비스

## 2. 배경 및 문제 정의

### 2-1. 기존 기획의 한계
기존 설계는 검증된 멘토, 공강 스케줄 입력, 오프라인/캠퍼스투어, 정산/패널티/등급화 중심 구조였고, 상품 포트폴리오도 1:1 멘토링, 프리미엄 캠퍼스투어, B2B 학교 단체투어 중심이었음. 아래 문제가 존재.

- 공강 시간대와 실제 학생/학부모 수요 시간대가 일치하지 않음
- 오프라인 참여는 이동 시간, 교통비, 일정 조율 부담이 큼
- 캠퍼스 방문 자체가 반복 수요가 큰 멘토링 상품이 되기 어려움
- 대학생이 직접 프로그램을 개설·운영하는 구조는 초기 품질 관리 비용이 큼
- MVP 단계에서 운영 리스크가 과도하게 큼

### 2-2. 회의 반영 핵심 판단
대표 미팅 기준으로 아래 방향을 최종 반영.

- 서비스는 온라인 면대면 멘토링으로 단일화
- 패키지 판매보다 멘토링 중개에 집중
- 탐색은 문제 해결형 구조로 설계
- MVP는 3개 대학 중심으로 시작
- 가격은 관리자 설정 범위 내 자율 (권장가 29,900원 기준으로 수요 검증)
- 알림은 카톡 기반 MVP로 시작
- 멘토링은 30분 기준 표준 플로우를 갖춘다

## 3. 목표

### 3-1. 비즈니스 목표
- 초기 수요 검증 가능한 온라인 멘토링 중개 구조 확보
- 검증된 멘토 확보를 통한 신뢰 기반 브랜드 형성
- 예약/결제/후기/정산까지 이어지는 최소 거래 구조 완성
- 반복 가능한 카테고리형 멘토링 상품 구조 정립

### 3-2. 사용자 목표

**학생/학부모**
- 내가 가진 고민에 맞는 멘토를 쉽게 찾고 싶다
- 실제 경험을 가진 대학생에게 현실적인 조언을 받고 싶다
- 복잡한 탐색 없이 빠르게 상담을 예약하고 싶다

**멘토**
- 검증된 플랫폼에서 안전하게 멘토링을 진행하고 싶다
- 공강/여유 시간에 효율적으로 수익을 얻고 싶다
- 후기와 평점을 쌓아 신뢰를 확보하고 싶다

### 3-3. MVP 성공 목표
- 검증 완료 멘토 풀 확보
- 첫 예약/결제 전환 달성
- 후기 데이터 축적
- 멘토링 카테고리별 초기 수요 파악
- 알림 누락 없는 운영 흐름 확보

## 4. 비목표 (MVP 범위에서 제외)

- 오프라인 캠퍼스투어 운영
- 학교 단체투어/B2B 단체 운영
- 자체 화상 채팅 시스템 개발
- AI 매칭 추천
- 학부모 자동 리포트 생성
- 복잡한 멘토 등급 자동화
- 증명서 자동 발급
- 앱 푸시 중심 알림 시스템
- 패키지 상품 직접 기획·운영

## 5. 타겟 사용자

### 5-1. 1차 타겟: 중·고등학생

주요 니즈
- 공부법이 맞는지 모르겠음
- 내신/수능 준비 방향이 궁금함
- 어떤 대학/전공이 나와 맞는지 알고 싶음
- 입시를 먼저 겪은 선배 얘기를 듣고 싶음
- 학생부/면접/진로 고민이 있음

### 5-2. 2차 타겟: 학부모

주요 니즈
- 자녀의 진로 방향을 외부 시각으로 점검하고 싶음
- 학원/입시 기관이 아닌 실제 경험자의 조언을 듣고 싶음
- 특정 학교/전공 현실을 알고 싶음
- 자녀에게 맞는 멘토를 대신 찾아주고 싶음

### 5-3. 공급자: 대학생 멘토

조건
- 재학생 또는 합격 경험 증빙 가능자
- 학교/전공/입시 경험 명시 가능자
- 온라인 상담 가능자
- 기본 응답 및 일정 준수 가능한 자

## 6. 핵심 가치 제안

### 학생/학부모에게
- 대학생의 실제 경험을 기반으로 한 현실적인 멘토링
- 문제 해결형 카테고리 탐색
- 온라인으로 편하게 예약/진행 가능
- 후기/검증 기반 선택 가능

### 멘토에게
- 신뢰 가능한 프로필 기반 활동 가능
- 수익화 가능
- 무작위 오픈마켓이 아닌 검수 기반 운영
- 일정 조율 및 정산 지원

## 7. 핵심 가설

- 학생/학부모는 "대학생 멘토링" 자체보다 "구체적 문제 해결" 중심 구조에서 더 쉽게 신청한다
- 오프라인보다 온라인 면대면 방식이 초기 전환율과 운영 효율 면에서 유리하다
- 학과를 세부 전공까지 모두 펼치기보다, MVP에서는 3개 대학 + 단과대 기반 구조가 탐색과 공급 관리 모두에 유리하다
- 29,900원은 UI 권장가(기본값)로 기능하며 강제 floor가 아님. 관리자 설정 최저가~최고가 범위 내에서 멘토가 직접 책정할 때 수요가 자연스럽게 분포한다 (ADR-003)
- 알림 시스템이 없으면 신청 누락과 노쇼가 발생해 운영 품질이 급격히 떨어진다
