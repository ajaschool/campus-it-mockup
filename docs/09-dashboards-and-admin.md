### 19-4. 운영자 수동 동선 (MVP 화면 미구현, ADR-002)

MVP에서는 다음 동선이 화면 없이 **운영자가 SQL/엑셀로 직접 처리**. 화면 자동화는 W5+.

| 동선 | 처리 방식 | 트리거 |
|---|---|---|
| **쿠폰 발급 (클레임 보상 한정)** | 운영자가 admin DB 콘솔 또는 엑셀 → SQL로 `coupons` row INSERT → 코드를 1:1 문의 답변(`inquiries.answer_body`) 또는 클레임 답변에 본문으로 전달 | 클레임 결과 보상 / 운영 사과 |
| **쿠폰 만료 처리** | 운영자가 `coupons.expires_at`을 발급 시점에 SQL로 직접 입력 (자동 만료 알림 없음) | 발급 시 |
| **노쇼 자동 판정 이의 (ADMIN_OVERRIDE)** | 1:1 문의로 접수 → `bookings.noshow_detection_source = ADMIN_OVERRIDE` SQL 변경 + 환불·패널티 사후 정정 | 사용자 이의 (docs/05 §20-3-5) |
| **약관 본문·버전 관리** | CMS 화면(§5.9.3) 사용. 단, MVP는 IN-수동이라 운영자가 직접 폼 입력 | 약관 개정 |
| **대학·단과대 시드 관리** | 시드 SQL → 점진적으로 관리자 콘솔로 이관 | 신규 대학 추가 |

**규칙**
- SQL 직접 변경 시 변경 이력은 운영팀 내부 노션·슬랙에 기록 (DB 자체 audit log는 W5+)
- 쿠폰 코드는 영문 대문자 + 숫자 8~12자 무작위 (운영자 자율, 중복은 PK 강제로 차단됨)
- 클레임 보상 쿠폰은 `coupons.issued_to_user_id`를 멘티 user_id로 명시해 1인 한정 사용

근거: docs/11 ADR-002 (§25-7 쿠폰 발급 OUT-W5 / 사용 IN-수동 클로즈)
